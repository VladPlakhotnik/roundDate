# Анимация главного экрана

## Контекст

В `references/main.png` вокруг заголовка стоят два кресла. Они работают как главный эмоциональный образ hero-секции: два места для разговора, офлайн-встреча, уютная атмосфера.

## Вариант 1: полноценное 3D

Смысл: кресла становятся настоящими 3D-моделями, которые слегка поворачиваются при движении мыши или скролле.

Плюсы:

- Самый дорогой и вау-эффектный вариант.
- Можно красиво вращать кресла, менять свет и делать parallax.
- Хорошо подойдет для будущего брендового hero.

Минусы:

- Нужно получить качественные модели.
- 3D увеличит вес страницы и усложнит оптимизацию.
- Для SEO-лендинга это может быть избыточно на MVP.

Как сделать модель из картинки:

1. Подготовить отдельные изображения кресел без фона.
2. Прогнать через сервис image-to-3D: Meshy, Tripo, Luma AI, Rodin или аналог.
3. Экспортировать в `glb`.
4. Оптимизировать модель: уменьшить polycount, сжать текстуры, проверить вес.
5. Подключить через Three.js или React Three Fiber.

Рекомендация: не делать это в первой итерации. Вернуться к 3D после утверждения визуального направления.

## Вариант 2: псевдо-3D из 2D-ассетов

Смысл: оставить кресла PNG/WebP, но добавить ощущение глубины через слои.

Анимации:

- легкий floating: кресло медленно поднимается и опускается на 6-10px;
- micro-tilt: при движении курсора кресло поворачивается на 2-4 градуса;
- depth shadow: тень слегка меняется при hover/scroll;
- parallax: левое и правое кресло двигаются с разной скоростью относительно заголовка.

Плюсы:

- Почти такой же визуальный эффект для пользователя.
- Намного проще и быстрее 3D.
- Легко контролировать производительность.
- Можно сделать уже на текущих ассетах.

Минусы:

- Нельзя честно вращать объект на 360 градусов.
- При сильном наклоне будет видно, что это плоская картинка.

Рекомендация: лучший вариант для MVP.

## Вариант 3: минимальная 2D-анимация

Смысл: сделать hero живым, но не отвлекать от CTA.

Анимации:

- кресла плавно появляются с небольшим scale и blur;
- декоративные звезды мягко пульсируют;
- орбитальная линия медленно смещается на 1-2 градуса;
- CTA получает очень тонкий glow при hover;
- карточки снизу появляются staggered-анимацией.

Плюсы:

- Самая безопасная реализация.
- Минимальный риск проблем на мобильных.
- Хорошо сохраняет точность к референсу.

Минусы:

- Меньше вау-эффекта, чем у 3D.

Рекомендация: внедрить первой, если дизайн будет утвержден.

## Итоговая рекомендация

Для текущего этапа не нужен настоящий 3D. Лучше сделать псевдо-3D: PNG/WebP-ассеты с мягким floating, parallax и mouse-tilt. Это сохранит стиль `main.png`, даст ощущение живой сцены и не перегрузит лендинг.

Настоящее 3D стоит делать позже, только если клиент прямо выберет этот hero как финальный брендовый образ.
